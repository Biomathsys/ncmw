Hierarchical model compostionÂ¶

This implements a way to compute Hierarchical metabolic models, according to the SBML âcompâ package. This has several advantages: * Within the SBML file only variables and constraints relevant for the community are saved. All other necessary variables stay within the original SBML files. * Changes to the individual models will be automatically transfered to the community model.

The constructed SBML file does not contain the discription of the individual models, but instead only describes how these models should be combined! As a result we save there only things like shuttle reactions.

[1]:

from ncmw.community import HierarchicalCommunityModel

[2]:

model = HierarchicalCommunityModel(["../data/models/iDPM21RW.xml", "../data/models/iYS854.xml"])

It does read the âdescriptionâ on how to combine the individual models and exactly does this putting everythink into one file. This can now be read by cobrapy. All this is done in the background, in the end we obtain the same commuity model as before.
